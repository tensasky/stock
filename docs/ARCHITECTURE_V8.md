## 二、数据流

```
1. 每日收盘后
   └─ 数据Agent抓取日线 → 本地数据库
   └─ 预计算技术指标 → indicators表

2. 交易时段(10:00/11:00/14:00)
   └─ 读取本地指标(昨日)
   └─ realtime_price获取实时价格
   └─ 策略筛选(技术面+实时涨幅)
   └─ 执行买入

3. 盘中监控
   └─ 实时价格监控
   └─ 止盈止损判断
   └─ 报告Agent发送通知
```

## 三、实时价格获取

### 多数据源容错
```
优先级: 腾讯API → 新浪API → 东方财富API → 缓存(5分钟) → 昨日收盘

容错机制:
- 每个源失败重试3次
- 获取成功缓存5分钟
- 全部失败返回过期缓存
```

### 交易前提条件
```
⚠️ 只有当日实时价格获取成功才执行交易

检查项:
□ realtime_price返回非空
□ 价格 > 0
□ 实时涨跌幅计算成功
```

## 七、关键规则

### ⚠️ 交易前置检查
```
1. 检查实时价格获取是否成功
2. 检查涨跌幅是否计算成功
3. 只有实时数据有效时才执行交易
4. 获取失败则跳过本次交易
```
